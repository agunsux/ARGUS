## Purpose

*Menangani dispute antara buyer dan seller.*

## Dispute Categories

| Category | Description |
|----------|-------------|
| Bukti tidak jelas | Evidence yang diupload tidak readable |
| Tiket tidak valid | Tiket ditolak di venue |
| Seller tidak hadir | Seller gagal transfer tiket |
| Double booking | Tiket dijual ke dua orang |
| Fraud | Indikasi penipuan |

## Resolution Flow

1. Buyer mengajukan dispute
2. Ops mengumpulkan evidence
3. Ops memverifikasi kronologi
4. Keputusan: refund / split / dismiss
5. Update status transfer

## Escalation

Jika tidak ada resolusi dalam 48 jam → eskalasi ke founder.
